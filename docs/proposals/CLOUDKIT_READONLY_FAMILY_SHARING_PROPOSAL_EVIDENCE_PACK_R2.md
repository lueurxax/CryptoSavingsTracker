## D. Assumptions and Open Questions
- ASSUMP-01: The proposal's new full-goal-set model intentionally accepts that future goals auto-share until access is revoked.
- ASSUMP-02: Xcode Preview evidence is sufficient for this repeat review because the feature remains proposal-only and the current-state shell is unchanged.
- ASSUMP-03: The proposal still targets one owner-to-many-invitees sharing model and not collaborative multi-writer household planning.
- QUESTION-01: How should `Shared Goals` be grouped when one invitee receives shared datasets from multiple different owners?
- QUESTION-02: Does the local read-only cache/store mean a separate SwiftData container, a separate SQLite file, or a non-SwiftData cache layer, and does that need to be locked at proposal level?
- QUESTION-03: What rollout thresholds or error-rate guardrails trigger the kill switch in practice?
